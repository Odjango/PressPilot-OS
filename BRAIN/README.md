# The BRAIN System
**Goal:** A "Designer-First" operating system where you control the logic via text files and images.

## 📂 Folder Structure

### 1. `CONSTITUTION/` (The Laws)
**Logic you never want to repeat.**
*   **`core_directives.md`**: Market rules (No Plugins, No Lock-in).
*   **`design_system.md`**: Aesthetic rules (No Hex codes, use Variables).
*   **`fse_laws.md`**: Technical rules (Paragraph Links in Footer).
*   **`verticals/`**: Definitions for specific industries (Fitness, Pizza, Shop).

### 2. `MEMORY/` (The History)
**What we have learned.**
*   **`project_state.md`**: The current "Save Game". Records solved problems (Logo Upload, Color Logic).
*   *Action:* If I forget something, point me here.

### 3. `VISION/` (The Input)
**Your Design Specs.**
*   *Action:* Drop screenshots, diagrams, or text files here.
*   *Example:* Drop `cool_footer_vibe.jpg` here and tell me "Make the footer look like `VISION/cool_footer_vibe.jpg`".

---
**How to use:**
1.  **To change a rule:** Edit the file in `CONSTITUTION/`.
2.  **To start a new feature:** Drop a spec in `VISION/`.
3.  **To save progress:** I will update `MEMORY/` automatically.
