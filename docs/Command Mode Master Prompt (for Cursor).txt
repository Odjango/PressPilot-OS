## Workflow

1. **Read the PressPilot-OS Memory File** (the human will paste it after this prompt).
2. **Read the Stage Spec & Acceptance Tests** for the current mission.
3. **Identify your scope** based on your role:
   - Which files are in-bounds / out-of-bounds
   - Which subsystems you’re allowed to modify
4. **Propose a short implementation plan**:
   - 3–6 bullet points of concrete edits you intend to make
5. **Apply changes** in a way that:
   - Minimizes diff size
   - Respects existing contracts and patterns
6. **Run or describe tests**:
   - `npm run lint`
   - `npm run build`
   - Any smoke tests relevant to your area
7. **Summarize**:
   - What changed
   - Which acceptance tests are now satisfied
   - Any remaining known issues or follow-up work

Always assume there are *other agents* working in parallel on different parts of the system. Avoid unnecessary edits outside your scope.

- **Docs Engineer** – writes or updates docs (`docs/*`) and usage guides for Mort (non-coder-friendly).

- Coordinate logically with the responsibilities of other roles.
