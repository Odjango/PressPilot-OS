# PressPilot Theme Generator Rules

This document defines the "Gold Standard" structure for themes generated by the PressPilot Theme Generator.

## 1. File Structure
- `functions.php`: Contains core logic for content setup and theme configuration.
- `theme.json`: Defines global styles, layout settings, and block defaults.
- `style.css`: Theme metadata (Name, Version, Author).
- `templates/`:
    - `index.html`: Fallback template (Blog).
    - `page.html`: Default page template (MUST allow full-width content).
    - `front-page.html`: Home page template (MUST allow full-width content).
    - `single.html`: Single post template.
    - `404.html`: Error page (Critical to prevent confusing fallbacks).
- `parts/`:
    - `header.html`: Contains navigation.
    - `footer.html`: Site footer.

## 2. Core Logic (`functions.php`)
### Separation of Concerns
- **Configuration**: Use `after_setup_theme` for persistent settings like `add_theme_support('menus')`. This MUST run on every request.
- **Content Setup**: Use `after_switch_theme` (and `admin_init` version check) for one-time content creation ("Nuke and Pave").

### Nuke and Pave Strategy
- **Delete**: Remove all existing pages, menus, and `wp_navigation` posts to ensure a clean slate.
- **Create**: Programmatically insert core pages (Home, About, Services, etc.).
- **Menu**:
    - Create `wp_navigation` post for FSE Site Editor.
    - Create Classic Menu ("Primary Menu") for Dashboard visibility.

## 3. Template Requirements
### Full Width Support
- `wp:post-content` blocks in `page.html`, `front-page.html`, and `single.html` MUST have:
  ```json
  {"align":"full","layout":{"type":"constrained"}}
  ```
  This allows inner blocks (like Cover/Hero) to span the full viewport width while keeping text constrained.

### 404 Handling
- Always include a `404.html` template. Without it, WordPress falls back to `index.html`, which often looks like the Blog page, confusing users.

## 4. Navigation
- **Hybrid Approach**: Support both FSE (Block) menus and Classic menus.
## 5. Library Assembly Strategy
The Generator must NOT synthesize themes from scratch. It must **Assemble** them from the Library:
- **Base Themes**: We strictly use vetted bases: Ollie, Frost, Twenty Twenty-Four.
- **Layout Recipes**: We use `PatternRegistry` to define "Recipes" (ordered list of patterns) for each industry (e.g. "SaaS Start", "Agency Portfolio").
- **Injection**: User content is injected *into* existing pattern files via text replacement, maintaining the vetted block markup.

## 6. Business Logic Triggers
The Generator executes logic based on the user's business type:

### Restaurant / Cafe
- **Trigger**: Industry = "restaurant".
- **Recipe**: Uses "Family Restaurant" recipe (Ollie).
- **Homepage**: Includes `patterns/menu-card-1.php` as a teaser.
- **Deep Page**: Generates a dedicated `/menu` page (`templates/page-menu.html`) and populates `patterns/presspilot-menu.php` with the full menu data.

### SaaS / Agency
- **Trigger**: Industry = "saas" or "agency".
- **Recipe**: Uses "SaaS Start" or "Agency Portfolio".
- **Action**: Focuses on Hero, Feature Grid, and Testimonials injection.

### Universal (Default)
- **Action**: Falls back to "Classic Business" recipe.

## 7. Enterprise-Grade Styling Rules
To ensure a premium, trustworthy aesthetic, all generated themes must adhere to:

### A. Color Harmonization
- **Saturation Cap**: Primary brand colors must be capped at 80% saturation.
- **Auto-A11y**: Brand color on white text must meet WCAG AA (4.5:1). If not, it receives an automatic darkening adjustment.
- **Headings Only**: The massive saturation of "primary" color should be restricted to **Headings (H1-H3)**.
- **Body Text**: Must strictly use a professional neutral (e.g., `#1E1E26`), NOT the brand color.

### B. Stable Navigation (FSE Compliance)
- **Rule**: `wp:navigation` blocks MUST be self-closing.
- **Prohibited**: Do NOT embed `wp:navigation-link` inner blocks in generated patterns. This causes "Attempt Recovery" errors in the Site Editor because there is no matching database record.
- **Structure**:
  ```html
  <!-- wp:navigation {"layout":{...}} /-->
  ```

### C. Consistent Branding
- **Footer**: Every theme footer must end with a unified line:
  > © 202X [Business Name]. All rights reserved. Powered by **PressPilot OS**
